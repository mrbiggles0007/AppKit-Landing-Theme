Requirements For A Remediation Project

Below we list the requirements for us to engage with your MSP company in a documentation remediation project, to achieve the best possible outcome.

  • Executive buy in - Project will fail without it.
  • Monthly review meetings with an executive or non technical senior staff member.
  • One IT Glue or Hudu Administrator license.
  • One Office365 license under your company with Multi-Tenant admin permission.
  • One PSA license with admin permissions such as Connectwise Manage.
  • One RMM user that we can access, ideally a named user license for auditing purposes.
  • Admin user accounts any other systems we need to work on.
  • Ability to enter time into the system used for internal staff (eg: Connectwise Manage)
  • Minimum 100 hour block for the review and completion of a meaningful set of outcomes.
